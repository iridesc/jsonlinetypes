# JsonLineTypes 实现原理 - 快速总结

## 🎯 核心概念

JsonLineTypes 让你像使用普通 dict/list 一样操作数据，但数据保存在磁盘的 JSONL 文件中，而不是 RAM 里。

## 📐 核心原理图

```
┌─────────────┐
│   JLFDict   │
└──────┬──────┘
       │
       ▼
┌────────────────────────────────────────┐
│  索引（内存中，~100 字节/记录）      │
│  _key_to_offset = {                   │
│    "user1": 0,    → 文件第 0 字节   │
│    "user2": 89,   → 文件第 89 字节  │
│  }                                      │
└───────┬──────────────────────────────┘
        │
        │ 索引查找（O(1)）
        ▼
┌────────────────────────────────────────┐
│  data.jsonl（磁盘上的 JSONL 文件）    │
│  {"id": 1, "name": "Alice"}           │
│  {"id": 2, "name": "Bob"}             │
│  {"id": 3, "name": "Charlie"}         │
└────────────────────────────────────────┘
```

## 🔑 关键技术

### 1. 文件偏移量索引

**不保存数据，只保存位置！**

```python
# 传统方式（内存大）
data = {1: {...}, 2: {...}, ...}  # 所有数据在内存

# JsonLineTypes 方式（内存小）
_key_to_offset = {1: 0, 2: 89, 3: 178}  # 只保存位置
```

### 2. 追加写入 + 软删除

**只追加，不修改原数据**

```
写入时：
[A, B, C]
[A, B, C, B(del), B_new]  ← 追加删除标记 + 新数据

删除时：
[A, B, C]
[A, B, C, B(del)]  ← 只追加删除标记

更新时：
[A, B]  
[A, B, B(del), B_new]  → 索引指向 B_new
```

### 3. JSONL 格式

**每行一个 JSON 对象**

```
{"id": 1, "name": "Alice"}   ← 第 0 字节
{"id": 2, "name": "Bob"}      ← 第 89 字节
{"id": 3, "name": "Charlie"}  ← 第 178 字节
```

## ⚡ 性能原理

### O(1) 查找

```python
# 索引查找（O(1)）
offset = _key_to_offset["user2"]  # 89

# 文件跳转（O(1)）
f.seek(89)

# 读取一行（O(1)）
line = f.readline()

# 总计：O(1)
```

### 内存占用

```
100万条记录，每条500字节

传统方式：500 MB RAM
JsonLineTypes：100 MB RAM（只保存索引）

节省：80% 内存
```

## 🔄 工作流程示例

### 写入

```python
d = JLFDict("data.jsonl", "id")
d[1] = {"id": 1, "name": "Alice"}

# 内部：
# 1. 追加到文件末尾
# 2. 记录偏移量（例如：0字节）
# 3. 更新索引：{1: 0}
# 4. 保存索引文件
```

### 读取

```python
value = d[1]

# 内部：
# 1. 查找偏移量：_key_to_offset[1] = 0
# 2. f.seek(0) 跳转
# 3. f.readline() 读取
# 4. json.loads() 解析
# 5. 返回数据
```

### 更新

```python
d[1] = {"id": 1, "name": "Alice2"}

# 内部：
# 1. 追加删除标记：{"id": 1, "_deleted": true}
# 2. 追加新数据：{"id": 1, "name": "Alice2"}
# 3. 更新索引指向新位置
```

### 删除

```python
del d[1]

# 内部：
# 1. 追加删除标记：{"id": 1, "_deleted": true}
# 2. 从索引移除
# 3. 添加到已删除集合
```

### Compact

```python
d.compact()

# 内部：
# 1. 创建临时文件
# 2. 只复制未删除的记录
# 3. 重写偏移量
# 4. 替换原文件
# 5. 重建索引
```

## 🛡️ 线程安全

**所有操作都受 threading.RLock 保护**

```python
# 保护的资源：
• _key_to_offset（字典）
• _deleted_keys（集合）
• 索引文件读写

# 性能开销：2-8%（很小，因为磁盘 I/O 是主要瓶颈）
```

## 💡 核心创新

1. **文件偏移量索引** - 只存位置，不存数据
2. **追加写入** - 不需要移动已有数据
3. **软删除** - 用标记代替实际删除
4. **延迟读取** - 按需加载数据
5. **索引持久化** - 快速启动，可重建

---

**一句话总结：JsonLineTypes 通过文件偏移量索引、追加写入和软删除等技术创新，实现了类 dict/list 接口的大规模数据存储，用极小的内存占用处理任意大小的数据集。**
